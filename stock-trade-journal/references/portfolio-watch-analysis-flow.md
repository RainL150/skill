# STJ 持仓与关注复盘流程

用于用户要求“分析我的持仓”“持仓盈利”“操作推荐”“分析持仓还有关注”“看看关注列表能不能买”等组合级请求。

这个流程是持仓和关注分析的默认入口。用户要求“分析”时必须直接产出分析结论；不要返回提示词，也不要用图表或报告替代。

## 0. 输出目标：交易决策版，不是体检版

组合级分析的目标是给出可执行交易计划，不是复述持仓、行情和 note。输出必须先回答：

- 组合当前状态：进攻 / 防守 / 换仓 / 降风险 / 等验证。
- 最大矛盾：集中度、逻辑失效、机会成本、现金不足、同逻辑拥挤、亏损拖累或兑现纪律缺失。
- 今日或本周最重要的 3 个动作，并按优先级排序。
- 如果今天只能加一个标的，加哪个；如果必须减一个标的，减哪个；如果都不该动，也要说明“为什么现金/等待优于交易”。

禁止把组合分析写成流水账。若只能得到本地 note 和行情、缺少外部验证，必须降低动作强度，并明确下一步要查什么数据才能升级为买入、加仓或退出。

## 1. 先读本地上下文

必须读取：

```bash
cd ~/.claude/skills/stock-trade-journal/scripts
python3 query_positions.py
python3 watchlist.py ls
```

如需判断交易节奏和最近加减仓，按标的补读：

```bash
python3 query_trades.py --ts-code <代码> --limit 5
python3 analyze_holdings.py context <代码或名称> --json
python3 profile_review.py single <代码> --json
```

读取 profile：

```text
profiles/<用户指定或当前适用的交易画像>.md
```

Profile 是可替换的外部输入，不属于本流程的固定内容。选择规则：

- 用户明确指定 profile 时，读取指定 profile。
- 用户说“按我的画像”“参考画像”但没有指定名称时，查看 `profiles/`；如果只有一个可用 profile，可作为当前画像读取。
- 如果有多个 profile 且用户没有指定，不能硬套某一个画像；应说明需要指定画像，或先按无画像流程输出并标注“未套用交易画像”。
- 不要把任何具体 profile 的规则复制进本流程；只引用 profile 文件里的必填字段、仓位上限、加仓/再买规则、失效条件和复盘要求。
- 必须引用 profile 文件里的“交易拦截器”。组合级动作建议不能绕过拦截器。

若交易记录没有写入所选 profile 要求的字段，不要编造当时的主逻辑；指出缺少哪些可复盘字段。

## 1.1. 必要的外部验证

组合级建议不能只依赖本地 note。除非用户明确只要求本地数据，否则必须按标的类型补足最低外部验证：

- 成长股：最近财报收入增速、利润率/现金流、管理层指引、估值是否仍能被增长消化。
- 困境反转：坏消息是否继续发酵、监管/诉讼/舆情进展、盈利修复是否出现。
- 周期/制造：订单、价格、库存、开工率、海外收入、利润率或行业景气变量。
- 商品/ETF：底层期货、供需、库存、天气、官方周报或发行方持仓说明。
- 题材/事件股：题材是否兑现、利好是否出尽、股价是否已经脱离基本面。

如果外部验证暂时无法完成，动作建议只能是“观察/小仓试错/等待/减风险”，不能强行给出高置信加仓。

## 2. 统一行情口径

不要混用零散网页片段当作精确行情。必须说明价格时间口径。

优先行情源：

1. A 股、港股、常见美股：东方财富 push2 行情接口或同等可核验行情库。
2. 商品/ETF 等东方财富取不到的标的：发行方页面、Yahoo Finance、交易所或可信报价页，并明确收盘日。
3. 汇率：仅用于组合权重粗估；必须标注“估算”，不要和本地原币种成本混为一个精确总额。

价格输出要求：

- A/H/美股分别保留原币种盈亏。
- 组合权重可以用人民币等值估算，但必须写明汇率口径。
- 如果美股是上个交易日收盘、A 股是盘中价，要明确区分。
- 若任一价格无法确认，标为“未确认”，不要用旧价硬算。

## 3. 按画像约束动作建议

如果加载了 profile，动作建议必须受该 profile 约束：

- 使用 profile 中定义的主逻辑分类、交易周期、失效条件、加仓/再买规则、仓位上限和复盘日期。
- 使用 profile 中定义的交易拦截器，先判断当前动作是符合、部分符合还是不符合画像纪律。
- 加仓、减仓、止损、策略切换只按 profile 中的条件判断；不要把某个固定画像的规则当成通用规则。
- 如果 profile 定义了仓位上限或同类暴露限制，先检查是否超限，再谈买入或加仓。
- 如果交易 note 缺少 profile 要求的字段，提示补齐，且不要把“被套持有”自动解释成新的长期逻辑。
- 如果没有可用 profile，使用普通投研和风险控制输出，但明确说明“未套用交易画像”。

每个重点持仓或候选动作必须先输出画像拦截摘要：

```text
当前动作：
画像匹配：
触发拦截器：
外部验证结果：
新证据：
仓位影响：
同公司/同逻辑合并暴露：
允许动作：
禁止动作：
需要补齐的 note 字段：
```

画像拦截摘要必须落到交易动作上。不能只列拦截器名称，必须说明该拦截器导致“允许/禁止/降级”的具体动作。

## 4. 组合级检查

必须覆盖：

- 当前持仓盈亏：数量、均价、当前价、市值、浮盈亏、收益率。
- 集中度：单一公司、同一 A/H 公司、同一行业、同一宏观因子、同一币种。
- 相关性：例如 AI/广告/互联网、周期制造、医疗监管、商品价格、汇率。
- 现金和机会成本：如果无法读到现金，就明确“未纳入现金”。
- 动作优先级：先处理高权重和高风险，再处理小仓波动。
- 组合模式：进攻、防守、换仓、降风险或等验证。
- 仓位约束：单一公司、同一逻辑、同一币种和单笔新开仓的建议上限；若 profile 没有硬参数，必须给“临时风控参数”，并标注为建议口径。
- 机会成本比较：当前持仓和关注列表中，谁最值得新增资金，谁最应该释放资金。

同一公司跨市场持仓必须合并看经济暴露，例如 A 股 + H 股、ADR + 港股。

## 5. 持仓逐一处理

每个持仓给出：

- 当前盈亏和组合权重。
- 主逻辑是否仍成立。
- 当前动作：持有、减仓、加仓、退出、仅观察或换仓。
- 仓位动作树：
  - 继续持有条件：价格结构、估值或基本面必须满足什么。
  - 加仓触发：必须同时包含价格/估值条件、基本面验证条件和仓位上限。
  - 减仓触发：包含可观察价格位、涨幅兑现、集中度超限或基本面验证不足。
  - 退出触发：包含失效价格位、财报/经营/事件证伪或复盘期限到期。
- 计划比例：如果建议减仓或加仓，必须给出相对仓位比例，例如减 1/3、减到组合 X% 以下、首笔 X%-Y%。无法精确计算时给区间。
- 下一次复盘点：财报日期、事件日期、价格触发位或固定日期。

持仓结论要和所选画像一致。若 profile 对仓位、止盈、止损、加仓或策略切换有明确约束，必须优先执行这些约束；不要用“看好”解释无限持有。

禁止只写“继续持有”“持有观察”“等待验证”。这些词只能作为动作标签，后面必须跟触发条件、仓位计划和失效条件。

## 6. 关注标的处理

关注列表是买入候选，不是持仓。必须结合关注笔记：

- 当前价格相对目标价/观察价的位置。
- 现在能不能开仓。
- 如果不能，等待什么触发条件。
- 首笔仓位建议和加仓规则。
- 失效条件。

关注列表必须排序，而不是逐个平均点评。至少分为：

- 今天可开仓：最多 1-2 个，必须给触发价、首笔仓位和不买条件。
- 等价格：当前高于用户目标价或赔率不足，必须给等待区间。
- 等事件/财报验证：当前不能买，等具体公告、财报或经营数据。
- 降级/删除关注：逻辑弱、价格错过、验证失败或与组合现有暴露重复。

对于“等价格回落 X”的关注笔记，如果当前价格显著高于 X，结论应是继续等待，除非出现新的基本面催化并说明为什么可以改规则。

## 7. 动作优先级和执行清单

输出必须包含一张执行清单：

```text
优先级 | 标的 | 当前动作 | 触发条件 | 仓位动作 | 失效条件 | 下次复盘
```

动作优先级规则：

1. 先处理组合级风险：同公司跨市场、单一逻辑、单一币种或高权重亏损。
2. 再处理已经到达止盈、止损、复盘期限或事件窗口的标的。
3. 再处理新增资金机会；没有显著高赔率时，明确建议等待。
4. 小仓波动不抢占高权重风险的优先级。

## 8. 输出格式

推荐顺序：

1. 一句话结论：先说组合状态、最大问题和第一优先动作。
2. 数据口径：列明行情时间、汇率估算、无法确认的数据。
3. 当前盈亏表：原币种盈亏 + 人民币等值粗权重。
4. 组合状态和机会成本：只能加一个选谁、必须减一个选谁、是否应等待。
5. 组合风险和动作优先级。
6. 画像拦截器和纪律缺口。
7. 持仓动作树。
8. 关注列表买入候选排序。
9. 执行清单。
10. 需要补齐的交易画像字段。
11. 数据来源链接或出处。

## 9. 质量自检

不要输出泛泛的“基本面良好、建议关注”。每个动作建议都要能被价格、财报或用户笔记复盘。

最终回复前必须自检：

- 是否给出了组合状态和最大矛盾。
- 是否给出了今日或本周最重要 3 个动作。
- 是否有“只能加一个/必须减一个”的机会成本判断。
- 每个重点持仓是否有继续持有、加仓、减仓、退出条件。
- 每个可交易关注标的是否有触发价、首笔仓位、加仓规则和失效条件。
- 是否说明了价格时间、汇率口径和未纳入现金。
- 是否明确哪些建议因为缺少外部验证而降级。

若以上任一项缺失，必须补齐后再输出。不能把不完整分析交给用户。
