# STJ 持仓与关注复盘流程

用于用户要求“分析我的持仓”“持仓盈利”“操作推荐”“分析持仓还有关注”“看看关注列表能不能买”等组合级请求。

这个流程是持仓和关注分析的默认入口。用户要求“分析”时必须直接产出分析结论；不要返回提示词，也不要用图表或报告替代。

## 1. 先读本地上下文

必须读取：

```bash
cd ~/.claude/skills/stock-trade-journal/scripts
python3 query_positions.py
python3 watchlist.py ls
```

如需判断交易节奏和最近加减仓，按标的补读：

```bash
python3 query_trades.py --ts-code <代码> --limit 5
python3 analyze_holdings.py context <代码或名称> --json
```

读取 profile：

```text
profiles/<用户指定或当前适用的交易画像>.md
```

Profile 是可替换的外部输入，不属于本流程的固定内容。选择规则：

- 用户明确指定 profile 时，读取指定 profile。
- 用户说“按我的画像”“参考画像”但没有指定名称时，查看 `profiles/`；如果只有一个可用 profile，可作为当前画像读取。
- 如果有多个 profile 且用户没有指定，不能硬套某一个画像；应说明需要指定画像，或先按无画像流程输出并标注“未套用交易画像”。
- 不要把任何具体 profile 的规则复制进本流程；只引用 profile 文件里的必填字段、仓位上限、加仓/再买规则、失效条件和复盘要求。

若交易记录没有写入所选 profile 要求的字段，不要编造当时的主逻辑；指出缺少哪些可复盘字段。

## 2. 统一行情口径

不要混用零散网页片段当作精确行情。必须说明价格时间口径。

优先行情源：

1. A 股、港股、常见美股：东方财富 push2 行情接口或同等可核验行情库。
2. 商品/ETF 等东方财富取不到的标的：发行方页面、Yahoo Finance、交易所或可信报价页，并明确收盘日。
3. 汇率：仅用于组合权重粗估；必须标注“估算”，不要和本地原币种成本混为一个精确总额。

价格输出要求：

- A/H/美股分别保留原币种盈亏。
- 组合权重可以用人民币等值估算，但必须写明汇率口径。
- 如果美股是上个交易日收盘、A 股是盘中价，要明确区分。
- 若任一价格无法确认，标为“未确认”，不要用旧价硬算。

## 3. 按画像约束动作建议

如果加载了 profile，动作建议必须受该 profile 约束：

- 使用 profile 中定义的主逻辑分类、交易周期、失效条件、加仓/再买规则、仓位上限和复盘日期。
- 加仓、减仓、止损、策略切换只按 profile 中的条件判断；不要把某个固定画像的规则当成通用规则。
- 如果 profile 定义了仓位上限或同类暴露限制，先检查是否超限，再谈买入或加仓。
- 如果交易 note 缺少 profile 要求的字段，提示补齐，且不要把“被套持有”自动解释成新的长期逻辑。
- 如果没有可用 profile，使用普通投研和风险控制输出，但明确说明“未套用交易画像”。

## 4. 组合级检查

必须覆盖：

- 当前持仓盈亏：数量、均价、当前价、市值、浮盈亏、收益率。
- 集中度：单一公司、同一 A/H 公司、同一行业、同一宏观因子、同一币种。
- 相关性：例如 AI/广告/互联网、周期制造、医疗监管、商品价格、汇率。
- 现金和机会成本：如果无法读到现金，就明确“未纳入现金”。
- 动作优先级：先处理高权重和高风险，再处理小仓波动。

同一公司跨市场持仓必须合并看经济暴露，例如 A 股 + H 股、ADR + 港股。

## 5. 持仓逐一处理

每个持仓给出：

- 当前盈亏和组合权重。
- 主逻辑是否仍成立。
- 动作：持有、减仓、加仓、退出、仅观察。
- 加仓条件：必须同时包含价格/估值条件和基本面验证条件。
- 减仓或退出条件：必须包含可观察价格位或基本面失效信号。

持仓结论要和所选画像一致。若 profile 对仓位、止盈、止损、加仓或策略切换有明确约束，必须优先执行这些约束；不要用“看好”解释无限持有。

## 6. 关注标的处理

关注列表是买入候选，不是持仓。必须结合关注笔记：

- 当前价格相对目标价/观察价的位置。
- 现在能不能开仓。
- 如果不能，等待什么触发条件。
- 首笔仓位建议和加仓规则。
- 失效条件。

对于“等价格回落 X”的关注笔记，如果当前价格显著高于 X，结论应是继续等待，除非出现新的基本面催化并说明为什么可以改规则。

## 7. 输出格式

推荐顺序：

1. 一句话结论：先说组合最大问题和优先动作。
2. 数据口径：列明行情时间、汇率估算、无法确认的数据。
3. 当前盈亏表。
4. 组合风险和动作优先级。
5. 持仓逐一处理。
6. 关注列表买入候选排序。
7. 需要补齐的交易画像字段。
8. 数据来源链接或出处。

不要输出泛泛的“基本面良好、建议关注”。每个动作建议都要能被价格、财报或用户笔记复盘。
