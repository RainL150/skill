# 交易录入 Profile：左侧机会型成长价值

用于“左侧低吸、分批建仓、成长/价值混合、需要防止摊平错误”的交易风格。

这个 profile 不只是录入问卷，也是一组交易前拦截器。咨询买入、加仓、减仓、继续持有或卖出时，必须先用本 profile 检查动作是否合规，再给交易建议。

## 触发方式

只有用户明确要求使用该 profile 时才启用，例如：

- `/stj 记录 ... 用左侧机会型 profile`
- `/stj 按我的画像记录 ...`
- `/stj 用交易画像录入 ...`

默认 `/stj 记录` 不启用本 profile。

## 录入前追问

启用 profile 后，若用户没有主动提供必填内容，必须先追问，再记录交易。这个 profile 采用轻量硬约束，不要求把每笔交易写成完整投研报告。

### 必填 4 项

1. **主逻辑**：我这次买/卖主要因为哪个核心假设？
2. **交易周期**：短线 / 中线 / 长线。
3. **失效条件**：什么情况说明我错了，必须认错或重审？
4. **加仓/再买规则**：跌了什么情况下能加，什么情况下不能加？

### 可选 4 项

1. **辅助逻辑**：除了主逻辑之外，还有哪些加分项？
2. **仓位上限**：这只标的最多占组合多少？
3. **目标价/观察价**：止盈、观察或重审价格。
4. **复盘日期**：什么时候必须重新看一遍？

## 适用场景

- 标的价格经历回撤、错杀、估值压缩或情绪冷却。
- 核心基本面没有恶化，甚至出现边际改善。
- 市场短期定价和中期经营事实可能出现错配。
- 可以设置清晰复盘窗口，且愿意接受“逻辑没验证就退出”。
- 适合先小仓试错，再用财报、经营数据、价格确认或事件兑现决定是否加仓。

## 禁用场景

- 只是因为价格涨了、错过买点或担心踏空而追。
- 只是因为亏损想摊低成本，没有新的基本面或价格确认。
- 原催化落空后，用新的故事无期限继续持有。
- 财报、现金流、监管、竞争格局或商业模式已经破坏。
- 单一标的、同一公司 A/H、同一行业或同一宏观逻辑暴露已经过高。

## 可复制好习惯

好习惯只有在交易记录、复盘笔记、买卖闭环或外部数据支持时才可升级为规则；证据不足时只作为待验证假设。`note` 只能证明当时如何想，不能单独证明习惯有效。

- **题材波段兑现后退出**：当买入逻辑是题材/事件波段，且价格已经快速兑现、基本面或价值跟不上时，允许按计划止盈退出。证据等级 A：603103.SH 已形成买入-加仓-清仓闭环，退出后价格回落，说明“题材兑现后不恋战”是当前最强可复制习惯。
- **先写主逻辑再交易**：保留“主逻辑、周期、失效条件、加仓规则”的录入纪律。证据等级 B：多笔交易 note 结构完整，但只有完成外部验证或买卖闭环后，才能升级为具体交易 playbook。
- **左侧先试仓，不一次打满**：当基本面未坏但价格仍未确认时，只允许试仓；确认后再加。证据等级 B/C：RDDT、UNH、NVDA 等有阶段性价格和基本面支持，但不能脱离估值、监管或财报验证。
- **策略切换显式记录**：如果原逻辑未兑现但决定继续持有，必须写 `holding_review`，说明原逻辑为什么失效、新逻辑是什么、新失效条件和复盘日期。证据等级 B/C：CORN 已补 `holding_review`，但商品轮动逻辑尚未被价格充分验证。

## 需要规避的坏习惯

坏习惯必须转成交易拦截器，而不是只写提醒。

- **错过买点后追加强势**：如果价格已经显著高于原计划买点，且没有新基本面证据，禁止一次性补足目标仓位。
- **亏损摊平**：价格下跌同时核心数据变弱、催化落空或失效条件触发时，禁止加仓。
- **短线失败改长线**：交易周期从短线/波段变成长线时，必须写策略切换笔记，否则视为逻辑漂移。
- **集中度失控**：单一公司、同一逻辑或同一风险因子暴露过高时，先处理仓位，再讨论加仓。
- **把 note 当验证结果**：只因为当时写了主逻辑，就把这笔交易归纳成好习惯或坏习惯；必须先验证结果、复盘和外部证据。
- **同一公司跨市场重复暴露**：A 股、H 股或不同账户持有同一公司/同一逻辑时，必须合并看仓位和风险，不能把它们当成分散配置。
- **商品轮动未确认就延长持有**：商品/农产品/周期 ETF 必须用底层期货、供需、库存、天气或官方周报验证；价格和基本面都未确认时，不能把“可能轮动”升级成长期逻辑。
- **浮盈后自动归因正确**：上涨可能来自市场 beta、题材、估值扩张或运气；只有当原 note 的核心证据被外部数据验证，才允许写成可复制习惯。

## 交易拦截器

- `note_verification_gate`: 交易 note 只作为待验证主张；未完成卖出闭环、后续 `holding_review` 或外部行情/财报/事件验证前，不得归纳为可复制好习惯或强拦截器。
- `external_validation_required`: 画像更新、加仓、继续持有或把某笔交易归纳成习惯前，必须先查外部数据；能查行情、财报、公告、事件、商品供需的数据时，不允许只写“待验证”。
- `fomo_after_missed_entry`: 当前价格明显高于原计划买点，且没有新增基本面证据时，只允许小仓确认或等待回踩，禁止追涨补仓。
- `no_new_evidence_no_add`: 加仓必须有新证据，证据可以是财报/经营数据增强、事件兑现、价格回踩确认或突破后回踩不破；只有“原逻辑不变”不够。
- `loss_averaging_block`: 价格下跌且核心指标、催化或行业数据变弱时，禁止摊平。
- `strategy_switch_requires_review`: 原逻辑未兑现但继续持有，必须写 `holding_review`，包含原逻辑失效原因、新主逻辑、新失效条件和复盘日期。
- `concentration_cap_review`: 单一公司、同一行业、同一宏观因子或同一币种暴露过高时，默认禁止继续加仓，除非先给出减仓/退出计划。
- `same_company_cross_market_exposure`: 同一公司 A/H、美股/港股、ETF/正股或同一风险因子必须合并计算暴露；合并后过高时，禁止用“另一个市场更便宜”作为继续加仓理由。
- `commodity_thesis_requires_underlying_confirmation`: 商品轮动、农产品、能源、金属类交易必须验证底层期货价格、供需、库存、天气或官方周报；底层未确认时禁止加仓，只能观察或按复盘期限退出。
- `theme_profit_take_playbook`: 题材波段交易如果已经出现大幅兑现、利好出尽或价格脱离基本面，优先执行止盈/减仓，不允许临时改成长线。
- `review_deadline_required`: 波段、中期或事件型交易必须有复盘期限；期限到达未验证时，不允许无条件延期。
- `invalid_if_fundamental_weakens`: 如果收入、利润率、现金流、核心运营指标或监管/竞争格局破坏主逻辑，不能再用“便宜”解释下跌。

## 交易前提醒卡

每次咨询交易前，先输出：

```text
当前动作：
画像匹配：符合 / 部分符合 / 不符合
触发的拦截器：
外部验证结果：
新证据：
仓位影响：
同公司/同逻辑合并暴露：
允许动作：
禁止动作：
必须写入的 note_type：
```

如果触发高风险拦截器，先提示纪律问题，再给交易建议。

## 持仓复盘规则

每条 `holding_review` 至少回答：

1. 原 `trade_decision` 是否仍成立。
2. 是否发生策略切换。
3. 如果切换，原逻辑为什么失效。
4. 新主逻辑和新失效条件。
5. 下一次复盘日期或退出期限。

没有复盘日期的“继续持有”不算合格复盘。

## 追问模板与答案样例

追问时不要只问抽象问题，要给用户可选的表达样例，帮助其快速落笔。

### 交易逻辑样例

可提示用户从以下类型里选，也可以自由补充：

- 估值便宜：PE/PB/FCF yield 已低于历史或同业合理区间。
- 增长超预期：收入、利润、订单、用户数、客座率、产能利用率等核心指标超预期。
- 困境反转：坏消息已反映在价格里，但经营数据开始改善。
- 周期底部：行业供需、价格、库存、开工率或资本开支接近周期低位。
- 情绪错杀：短期事件导致下跌，但中长期利润逻辑没有被破坏。
- 质量折价：好公司因短期扰动跌到可接受估值。
- 事件催化：回购、分红、政策、产品、航线/产能释放、业绩预告等可能改善预期。

推荐追问句：

```text
你买/卖它的主逻辑是哪一类？例如：估值便宜、增长超预期、困境反转、周期底部、情绪错杀。请补一句能验证的核心假设。
```

### 失效条件样例

失效条件必须能观察、能复盘。可提示用户用以下句式：

- 如果收入增速低于 X%，说明增长假设失效。
- 如果利润率连续 N 个季度恶化，说明成本或竞争压力超预期。
- 如果经营现金流明显跟不上利润，说明盈利质量要重审。
- 如果政策落地不利，直接压缩业务空间或估值中枢。
- 如果核心经营指标转弱，例如客座率、ARPU、订单、销量、开工率、利用率下滑。
- 如果股价跌破关键位置，同时基本面也变坏，不再用“便宜”解释下跌。
- 如果原本期待的催化没有兑现，或兑现后价格不涨反跌，需要重审。

推荐追问句：

```text
什么情况出现你会承认这笔交易逻辑错了？请尽量写成可观察条件，例如收入增速低于 X、利润率恶化、政策不利、股价跌破且基本面也坏。
```

### 加仓/再买规则样例

加仓必须区分“价格跌了但逻辑更强”和“价格跌了且逻辑变弱”。

可以加仓的例子：

- 价格下跌，但最新财报、订单、运营数据或现金流继续改善。
- 价格下跌只是市场系统性回撤，个股核心假设没有变坏。
- 估值更便宜，同时失效条件没有触发。
- 价格回踩关键支撑后企稳，且基本面验证更强。

不能加仓的例子：

- 价格下跌，同时收入、利润率、现金流或核心运营指标恶化。
- 下跌来自政策、监管、竞争格局或商业模式被破坏。
- 原本买入逻辑依赖的催化落空。
- 已经超过该标的或该行业的仓位上限。

推荐追问句：

```text
如果买入后价格下跌，什么情况下你允许自己加仓？什么情况下即使跌了也不能加？
```

### 策略切换规则

短线/中线失败后，不允许自动改成长线。允许切换，但必须补一条“切换笔记”，回答：

1. 原逻辑为什么失效？
2. 新的主逻辑是什么？
3. 新的交易周期是什么？
4. 新的失效条件是什么？

推荐追问句：

```text
这是策略切换，不是普通持有。请补一句：原逻辑为什么失效？新主逻辑和新失效条件是什么？
```

## 写入格式

将答案合并到 `record_trade.py --note` 中，不新增数据库字段，不修改主流程参数。

推荐格式：

```text
<原始备注>；主逻辑: <答案>；周期: <短线/中线/长线>；失效条件: <答案>；加仓规则: <答案>
```

如果有可选项，追加：

```text
辅助逻辑: <答案>；仓位上限: <答案>；目标价/观察价: <答案>；复盘日期: <答案>
```

如果是策略切换，追加：

```text
策略切换: 原逻辑 <答案>；新主逻辑 <答案>；新周期 <答案>；新失效条件 <答案>
```

## 质量要求

- 必填只问 4 项，不要把录入变成重型问卷。
- 失效条件必须是可观察信号，不接受“基本面变差”“走势不好”这类空泛表达。
- 加仓规则必须区分“价格跌但逻辑更强”和“价格跌且逻辑变弱”。
- 策略切换必须写切换笔记，不能把“被套”自动改名为“长线”。
- 画像更新必须先通过 `note_verification_gate`：逐条验证 note 是否有买卖闭环、holding_review 或外部数据支撑。
- 画像更新不能停在本地审计；凡是能查的外部数据都必须查完，再把样本升级、降级或保留为待未来验证。
- 全局画像更新才可以修改整体 profile；单标的复盘只能生成 profile evidence，不能单独改写整体画像。
- 如果用户无法回答，可以记录为“待补充”，但要在最终回复中提醒该交易缺少完整决策约束。
