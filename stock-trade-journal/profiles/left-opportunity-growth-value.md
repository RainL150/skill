# 交易录入 Profile：左侧机会型成长价值

用于“左侧低吸、分批建仓、成长/价值混合、需要防止摊平错误”的交易风格。

## 触发方式

只有用户明确要求使用该 profile 时才启用，例如：

- `/stj 记录 ... 用左侧机会型 profile`
- `/stj 按我的画像记录 ...`
- `/stj 用交易画像录入 ...`

默认 `/stj 记录` 不启用本 profile。

## 录入前追问

启用 profile 后，若用户没有主动提供必填内容，必须先追问，再记录交易。这个 profile 采用轻量硬约束，不要求把每笔交易写成完整投研报告。

### 必填 4 项

1. **主逻辑**：我这次买/卖主要因为哪个核心假设？
2. **交易周期**：短线 / 中线 / 长线。
3. **失效条件**：什么情况说明我错了，必须认错或重审？
4. **加仓/再买规则**：跌了什么情况下能加，什么情况下不能加？

### 可选 4 项

1. **辅助逻辑**：除了主逻辑之外，还有哪些加分项？
2. **仓位上限**：这只标的最多占组合多少？
3. **目标价/观察价**：止盈、观察或重审价格。
4. **复盘日期**：什么时候必须重新看一遍？

## 追问模板与答案样例

追问时不要只问抽象问题，要给用户可选的表达样例，帮助其快速落笔。

### 交易逻辑样例

可提示用户从以下类型里选，也可以自由补充：

- 估值便宜：PE/PB/FCF yield 已低于历史或同业合理区间。
- 增长超预期：收入、利润、订单、用户数、客座率、产能利用率等核心指标超预期。
- 困境反转：坏消息已反映在价格里，但经营数据开始改善。
- 周期底部：行业供需、价格、库存、开工率或资本开支接近周期低位。
- 情绪错杀：短期事件导致下跌，但中长期利润逻辑没有被破坏。
- 质量折价：好公司因短期扰动跌到可接受估值。
- 事件催化：回购、分红、政策、产品、航线/产能释放、业绩预告等可能改善预期。

推荐追问句：

```text
你买/卖它的主逻辑是哪一类？例如：估值便宜、增长超预期、困境反转、周期底部、情绪错杀。请补一句能验证的核心假设。
```

### 失效条件样例

失效条件必须能观察、能复盘。可提示用户用以下句式：

- 如果收入增速低于 X%，说明增长假设失效。
- 如果利润率连续 N 个季度恶化，说明成本或竞争压力超预期。
- 如果经营现金流明显跟不上利润，说明盈利质量要重审。
- 如果政策落地不利，直接压缩业务空间或估值中枢。
- 如果核心经营指标转弱，例如客座率、ARPU、订单、销量、开工率、利用率下滑。
- 如果股价跌破关键位置，同时基本面也变坏，不再用“便宜”解释下跌。
- 如果原本期待的催化没有兑现，或兑现后价格不涨反跌，需要重审。

推荐追问句：

```text
什么情况出现你会承认这笔交易逻辑错了？请尽量写成可观察条件，例如收入增速低于 X、利润率恶化、政策不利、股价跌破且基本面也坏。
```

### 加仓/再买规则样例

加仓必须区分“价格跌了但逻辑更强”和“价格跌了且逻辑变弱”。

可以加仓的例子：

- 价格下跌，但最新财报、订单、运营数据或现金流继续改善。
- 价格下跌只是市场系统性回撤，个股核心假设没有变坏。
- 估值更便宜，同时失效条件没有触发。
- 价格回踩关键支撑后企稳，且基本面验证更强。

不能加仓的例子：

- 价格下跌，同时收入、利润率、现金流或核心运营指标恶化。
- 下跌来自政策、监管、竞争格局或商业模式被破坏。
- 原本买入逻辑依赖的催化落空。
- 已经超过该标的或该行业的仓位上限。

推荐追问句：

```text
如果买入后价格下跌，什么情况下你允许自己加仓？什么情况下即使跌了也不能加？
```

### 策略切换规则

短线/中线失败后，不允许自动改成长线。允许切换，但必须补一条“切换笔记”，回答：

1. 原逻辑为什么失效？
2. 新的主逻辑是什么？
3. 新的交易周期是什么？
4. 新的失效条件是什么？

推荐追问句：

```text
这是策略切换，不是普通持有。请补一句：原逻辑为什么失效？新主逻辑和新失效条件是什么？
```

## 写入格式

将答案合并到 `record_trade.py --note` 中，不新增数据库字段，不修改主流程参数。

推荐格式：

```text
<原始备注>；主逻辑: <答案>；周期: <短线/中线/长线>；失效条件: <答案>；加仓规则: <答案>
```

如果有可选项，追加：

```text
辅助逻辑: <答案>；仓位上限: <答案>；目标价/观察价: <答案>；复盘日期: <答案>
```

如果是策略切换，追加：

```text
策略切换: 原逻辑 <答案>；新主逻辑 <答案>；新周期 <答案>；新失效条件 <答案>
```

## 质量要求

- 必填只问 4 项，不要把录入变成重型问卷。
- 失效条件必须是可观察信号，不接受“基本面变差”“走势不好”这类空泛表达。
- 加仓规则必须区分“价格跌但逻辑更强”和“价格跌且逻辑变弱”。
- 策略切换必须写切换笔记，不能把“被套”自动改名为“长线”。
- 如果用户无法回答，可以记录为“待补充”，但要在最终回复中提醒该交易缺少完整决策约束。
