# STJ 跨市场投资数据看板与全局问 AI

## 1. Goal

把 `stock-trade-journal` 的在线持仓页升级为一个本地优先、覆盖 A/港/美股的投资工作台：交易台账仍是事实底座，页面补齐持仓与关注、STJ 原版 K 线记录、个股基本面、资讯雷达、每日复盘、板块知识，以及忠实参考 Vibe 的多供应商问 AI 能力。

最终体验不是把 Vibe 整站搬过来，而是让用户围绕自己的持仓、关注、交易纪律和研究知识，在任意页面拿到同口径数据并继续追问。

## 2. User Outcomes

- 打开首页即可判断组合暴露、权重、收益和需要处理的风险，不再在多个脚本间拼信息。
- 从持仓或关注进入同一套个股详情，图表、交易、笔记、业务、财务和资讯不割裂。
- 每日复盘能同时观察 A/港/美市场，但清楚区分真实资金流与表现代理。
- 能自己维护板块、产业链、核心知识和风险信号，形成长期可复用的研究上下文。
- 在任何页面打开问 AI，选择订阅 CLI 或自带 API，并让 AI 使用当前页面和可信数据作答。
- 外部数据失败、过期或缺失时能看见真实状态，不会把空值、旧值或替代指标误当实时事实。

## 3. Scope

### 3.1 全局框架与导航（NAV）

- `NAV-01`：桌面端使用固定左侧菜单，包含持仓、关注、每日复盘、资讯雷达、板块知识、问 AI 设置。
- `NAV-02`：移动端菜单可收起，核心列表、详情和 AI 抽屉可用，不产生横向不可控溢出。
- `NAV-03`：所有一级页面和个股详情都提供同一个 Ask AI 入口，并自动带上当前页面、市场、标的或板块上下文。
- `NAV-04`：保留现有 `/chart`、`/charts/*` 与 `/api/data` 行为，已有链接在升级后仍可使用。
- `NAV-05`：加载、空态、部分失败、过期数据和完全失败分别有明确视觉状态。

### 3.2 持仓与关注（PORT）

- `PORT-01`：持仓表增加人民币估算权重；总收益金额和收益率合并为一个单元格。
- `PORT-02`：删除“最大权重”指标，顶部保留总市值、总浮盈、今日盈亏和待处理提醒。
- `PORT-03`：权重以同一时点报价和汇率计算，并展示数据时间；同公司跨市场暴露可被识别。
- `PORT-04`：持仓和关注行均可打开同一套个股详情。
- `PORT-05`：关注详情展示关注状态、目标价、止损价和距目标价；没有持仓时不伪造成本或盈亏。
- `PORT-06`：盈亏金额与比例合并展示，遵循产品现有“涨红跌绿”，正负、零值和缺失值均有独立样式。

### 3.3 个股详情（STOCK）

- `STOCK-01`：详情包含概览、K线·记录、财报、资金面、期权、资讯·研报；不再出现独立“我的记录”栏目。
- `STOCK-02`：顶部以数量、成本、现价、浮动盈亏四格展示；关注标的没有持仓时转换为关注摘要。
- `STOCK-03`：概览说明主营产品/收入来源、竞争位置、客户或渠道与关键经营变量，并标注来源与日期。
- `STOCK-04`：K 线最大程度保持原 STJ 体验：曲线/K线切换、十字提示、缩放、时间区间、B/S/T 圆点、“记”菱形点、现价/均价/目标/止损线、统计胶囊。
- `STOCK-05`：K 线下直接铺开“交易标注”和“关注记录”两栏，笔记悬浮和下方详情一致；不得另做重复记录数据源。
- `STOCK-06`：财报用年度/季度柱状图展示营收、净利润、自由现金流，并补充毛利率、现金含量、研发、EPS 修正、资产负债、营运资金、应收和库存等可得质量指标。
- `STOCK-07`：不同财务周期和币种不得混画；不可得指标显示缺失原因，不显示 0。
- `STOCK-08`：资讯·研报合并展示新闻、研报和监管披露，支持类型筛选、原文跳转、时间与来源追踪。
- `STOCK-09`：资金面与期权只在市场和标的确有可靠数据时出现；不适用时说明原因。

### 3.4 资讯雷达（RADAR）

- `RADAR-01`：汇总全部持仓和关注标的的新闻、研报与监管披露，另提供独立 Investment News。
- `RADAR-02`：默认结合持仓权重、关注优先级、时效、相关度与风险标签排序，并允许按市场、范围、类型筛选。
- `RADAR-03`：每条内容展示关联标的、来源、发布时间、抓取时间和原文链接；去除明显重复项。
- `RADAR-04`：风险信号可以突出，但必须能追溯到原文，不把 AI 摘要当事实来源。

### 3.5 每日复盘（REVIEW）

- `REVIEW-01`：保留 Vibe 风格的大盘指数、全球市场、市场温度与轮动视图。
- `REVIEW-02`：轮动视图按 A股/港股/美股标签切换，标出指标日期与市场交易状态。
- `REVIEW-03`：A 股真实主力净流入标记为“资金流”；港美使用行业/ETF 表现时明确标记“表现代理”，不能共用误导性标题。
- `REVIEW-04`：休市、跨时区与数据尚未更新时，显示对应市场的 `as_of`，不把昨日数据当今日实时数据。

### 3.6 板块知识（SECTOR）

- `SECTOR-01`：用户可以新建、编辑、归档板块，维护标签、摘要、核心知识和关联标的。
- `SECTOR-02`：产业链支持上游/中游/下游节点及节点关系，可表达关键依赖与卡脖子环节。
- `SECTOR-03`：核心知识记录标题、类型、正文、来源 URL 和资料日期；没有来源的内容明确标为个人判断。
- `SECTOR-04`：板块页常驻四个 AI 问题：“按七维框架拆解”“有什么风险信号”“这个板块的产业链地图”“哪个环节卡脖子”。
- `SECTOR-05`：板块知识可作为全局 Ask AI 上下文，但 AI 不能无确认地改写板块或交易数据。

### 3.7 问 AI 配置与交互（AI）

- `AI-01`：完整参考 Vibe 的两类接入：订阅 CLI 和自带 API；同一时间只有一个当前配置。
- `AI-02`：订阅 CLI 支持 Claude Code、Qwen Code、DeepSeek CLI、Codex；OpenCode、Cursor Agent、Kimi 显示“即将支持”，不可伪装成可用。
- `AI-03`：API 预设覆盖 DeepSeek V4 Flash/Pro、SiliconFlow DeepSeek V3、OpenAI GPT-4o、MiniMax M2、豆包 Pro、OpenRouter GPT-4o、Groq Llama 3.3、Together Llama、MiMo 和任意 OpenAI-compatible 端点。
- `AI-04`：API 配置允许编辑 Base URL、Model、API Key；预设自动填默认值，豆包支持 `ep-…` 接入点，自定义配置支持本地网关/Ollama。
- `AI-05`：模型配置和 API Key 只存当前浏览器；后端不得持久化、打印或写入研究记录。
- `AI-06`：API 模式提供流式回复和受控 function calling；订阅 CLI 流式回复但不调用工具，由服务端预装页面与 STJ 上下文。
- `AI-07`：面板支持多轮历史、停止请求、错误恢复、建议问题、来源/工具轨迹和“存入研究记录”。
- `AI-08`：默认回答结构支持结论、证据、反证/风险和交易纪律；用户可在设置中调整默认挂载范围。
- `AI-09`：可选后端访问密钥命名为 `STJ_API_KEY`；配置页提供设置状态和清除浏览器配置入口。
- `AI-10`：AI 只拥有只读数据工具；任何板块/笔记/研究记录写入都必须由明确 UI 动作触发。

### 3.8 数据可信度与非功能要求（DATA/NFR）

- `DATA-01`：A/港/美统一使用可识别的 `ts_code`、市场、交易所和币种；数字字段保持数字类型，缺失为 `null`。
- `DATA-02`：外部数据带来源、原始数据时间、抓取时间、缓存状态与警告；过期回退不可静默。
- `DATA-03`：外部数据源按各自限制节流，尤其所有东财请求必须共享限速与重试策略。
- `DATA-04`：部分来源失败不拖垮整页；成功模块可展示，失败模块可单独重试。
- `NFR-01`：继续采用本地 Node 服务、vendored Alpine.js 与 ECharts，运行时零前端构建、无 CDN 依赖。
- `NFR-02`：默认只监听 loopback；自定义 AI Base URL 经过 URL 校验和 SSRF 防护。
- `NFR-03`：常规缓存命中下，组合页首屏目标 1 秒内返回本地壳和事实数据；慢外部模块渐进加载。
- `NFR-04`：所有新增 Python CLI 支持 `--help`、结构化 JSON、非零错误码和显式 workspace；Node 子进程可取消、可超时、无僵尸进程。
- `NFR-05`：数据规范化、缓存、迁移、路由、AI 流协议和关键页面具有离线 fixture 测试；默认测试不依赖公网。
- `NFR-06`：交付后的 skill 自包含，不能在运行时读取 Vibe 或 sibling skill 目录。

## 4. Resolved Decisions

- 本任务覆盖完整数据看板，但按可回滚里程碑实施；不拆成多个 Trellis 子任务，因为 UI、数据契约和 AI 上下文共享同一条强依赖链。
- 前端采用 Node SSR 壳 + vendored Alpine.js + JSON API，保留零构建；不引入 React/Vite/FastAPI。
- K 线以现有 STJ 模板为兼容契约，正式实现优先复用，不重新画一个近似版本。
- 数据优先级是跨市场底座先行；个股深数据按真实持仓覆盖推进，同时尽早交付 A 股复盘和资讯能力。
- 板块知识需要持久化，因此允许在 `db_schema.py` 中增加前向兼容的新表，但不改变既有交易、持仓、关注和笔记语义。
- “AI 数据工具”指只读、参数受控的结构化查询，不是第三种 AI 配置，也不能执行任意 URL、SQL 或 shell。
- 港美轮动缺少可靠净流入口径时显示表现代理；宁可缺失也不伪造可比数据。

## 5. Out of Scope

- 自动交易、券商下单、AI 自动修改持仓、关注、板块或纪律。
- Vibe 全站复刻、React/FastAPI 重建、云端账号、多租户和跨设备同步。
- 首版多模型并行回答、模型自动择优或后台长期保存 API Key。
- 把两个数据 skill 的全部端点无差别复制进来；只实现页面和 AI 工具消费的数据。
- 未经验证的数据源、需要绕过登录/付费墙的抓取，以及无法给出来源时间的数据。
- 首版承诺港股期权或港美真实行业净资金流。

## 6. Acceptance Criteria

### 用户体验

- [ ] `NAV-01..05`：左侧导航、移动适配、全局 Ask AI、旧路由兼容和五类状态均通过人工验收。
- [ ] `PORT-01..06`：真实组合上展示权重、合并收益、四项 KPI；持仓和关注均能打开详情，盈亏颜色正确。
- [ ] `STOCK-01..09`：至少用 A、港、美各一个标的验证详情；K 线与原 STJ 交互逐项对照通过。
- [ ] `RADAR-01..04`：持仓、关注、Investment News 可筛选，来源可追溯，重复内容被合并。
- [ ] `REVIEW-01..04`：A/港/美复盘可切换，资金流和表现代理标签准确，休市时间不误导。
- [ ] `SECTOR-01..05`：能完成“新建板块→维护标签/链路/知识→关联标的→用四个固定问题提问→归档”。
- [ ] `AI-01..10`：至少一个 CLI 和一个 OpenAI-compatible API 完成流式问答；API 工具轨迹可见，CLI 上下文正确，停止与错误恢复有效。

### 数据与工程

- [ ] A/港/美代码、币种、财务周期、缺失值、来源时间和缓存状态满足 `DATA-01..04` 的契约测试。
- [ ] 断网、单一 provider 失败、全部 provider 失败、过期缓存和限流场景均有自动测试或可重复 smoke 记录。
- [ ] 新表迁移在空库和现有库副本上通过，既有交易/持仓/关注/笔记行数与语义不变。
- [ ] API Key 不出现在 SQLite、缓存、日志、进程参数、研究记录和测试快照中。
- [ ] Python、Node、skill validator 与人工浏览器验收全部通过；无 CDN、无 Vibe/sibling skill 运行时依赖。
- [ ] 兼容模式可恢复旧首页，回滚不要求删除新表，也不会丢失既有数据。
